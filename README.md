# Offline Growth (Fabric 1.21.10)

Мод для **Fabric 1.21.10**, который позволяет растениям и детёнышам мобов дозревать, даже если чанки были выгружены.

## Мультиплеер

Поддержка мультиплеера заложена **на стороне сервера**:

- логика работает через `ServerChunkEvents` и `ServerEntityEvents`;
- catch-up симуляция выполняется на сервере, поэтому результат общий для всех игроков;
- мод подходит для **dedicated server** и для хоста в LAN/одиночной игре, где есть интегрированный сервер;
- клиентской синхронизации через кастомные пакеты не требуется, потому что рост применяется обычными серверными изменениями мира и сущностей.

## Как это работает

Ванильный Minecraft не тикает выгруженные чанки. Этот мод не пытается держать их активными — вместо этого он:

1. запоминает игровое время, когда чанк/сущность выгрузились;
2. при следующей загрузке считает, сколько тиков прошло;
3. применяет **catch-up simulation**:
   - для растений — приближённо досчитывает случайные тики;
   - для детёнышей мобов — прибавляет прошедшие тики к их возрасту.

Такой подход намного безопаснее для TPS, чем «вечная прогрузка» чанков.

## Что поддерживается

### Мобы
- все `AgeableMob` (коровы, овцы, свиньи, курицы, лисы, ламы, волки и т.д.)

### Растения / блоки
- пшеница, морковь, картофель, свёкла и другие `CropBlock`
- арбузы / тыквы (`StemBlock`)
- саженцы деревьев
- пропагулы мангровых деревьев
- какао
- сладкие ягоды
- адский нарост
- питчер-кроп
- бамбук
- кактус
- сахарный тростник
- растущие лианы / kelp через `GrowingPlantHeadBlock`
- torchflower crop
- chorus flower

## Ограничения

- Рост растений симулируется **приближённо**, а не абсолютно побитово как при реальных тиках чанка.
- Для очень долгого отсутствия число догоняющих random tick попыток ограничено сверху, чтобы не положить сервер на больших фермах.
- Мод не «оживляет» редстоун, печки, воронки и другие механизмы. Он касается только роста/созревания.
- Если сам сервер остановлен, игровой `gameTime` не идёт, поэтому мод не симулирует время при полностью выключенном сервере.

## Сборка

В этой среде проект удалось подготовить как исходники, но не удалось собрать бинарный `.jar`, потому что здесь нельзя подтянуть бинарные зависимости Gradle/Fabric из сети. Локально собрать можно обычной командой:

```bash
gradle build
```

Готовый jar окажется в:

```bash
build/libs/
```


## Автосборка через GitHub Actions

Я добавил workflow `.github/workflows/build.yml`.

Как получить готовый `.jar`, не ставя Gradle вручную:

1. Загрузи эту папку в новый GitHub-репозиторий.
2. Открой вкладку **Actions**.
3. Запусти workflow **Build Fabric Mod** вручную или просто сделай push в `main`.
4. После завершения сборки скачай artifact `offline-growth-fabric-1.21.10` — внутри будет готовый jar.

Это самый надёжный путь для этой среды, потому что GitHub runner сам скачает зависимости Fabric/Gradle из сети.

## Установка

### Для сервера
1. Установи **Fabric Loader** для Minecraft **1.21.10**.
2. Положи собранный jar мода в папку `mods` сервера.
3. Установи **Fabric API**.
4. Запусти сервер.

### Для одиночной игры / LAN
1. Установи Fabric Loader.
2. Положи jar мода в клиентскую папку `mods`.
3. Установи Fabric API.
